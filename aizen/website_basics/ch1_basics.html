Chapter 1: HTML Basics

Welcome to the first chapter of our web development course! Here, we'll learn the fundamentals of HTML, including elements, attributes, and basic structure.

HTML Basics: Structuring Web Content

HTML stands for HyperText Markup Language. Don't worry too much about the full name! Just know that HTML uses special keywords called tags to tell the web browser how to structure the content you want to display (like headings, paragraphs, images, links).

Tags usually come in pairs: an opening tag (like <p> for a paragraph) and a closing tag (like </p>, notice the slash /). The content you want to show goes *between* these tags.

Your First HTML Page: Basic Document Structure

Every single HTML page needs this basic skeleton structure to work correctly in a browser:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
    <!-- Links to CSS or other metadata go here -->
</head>
<body>
    <!-- Visible page content goes here -->
    <h1>Hello World!</h1>
    <p>This is a paragraph.</p>
</body>
</html>
  • <!DOCTYPE html>: Declares the document type.
  • <html>: The root element of the page.
  • <head>: Contains meta-information (like character set, viewport settings, title, links to stylesheets). Not directly visible on the page.
  • <body>: Contains the visible content of the page (text, images, links, etc.).

Think of the <head> as the "behind-the-scenes" info for the browser, and the <body> as the actual stuff the user sees on the page.

Common HTML Building Blocks (Tags)

Here are some of the most frequently used HTML tags to build your content:

  • <h1> to <h6>: Heading tags. <h1> is the main heading (like a chapter title), <h2> is a sub-heading, and so on down to <h6>. Use them to create a logical outline for your content.
  • <p>: Paragraph tag. Use this for regular blocks of text.
  • <a>: Anchor tag, used to create links. It needs the href attribute (see below) to know *where* to link to. Example: <a href="https://www.google.com">Go to Google</a>.
  • <img>: Image tag. Used to display images. It needs the src attribute (see below) to know *which* image file to show. It also needs an alt attribute for descriptions. This is a self-closing tag (doesn't need a separate </img> in modern HTML). Example: <img src="dog.jpg" alt="A picture of my dog">.
  • <ul>: Unordered list tag. Creates a bulleted list. You put list items (<li>) inside it.
  • <ol>: Ordered list tag. Creates a numbered list. You also put list items (<li>) inside it.
  • <li>: List item tag. Represents a single item within a <ul> or <ol> list.
  • <div>: Division tag. A generic container often used to group other elements together for styling or layout purposes (we'll use this a lot with CSS).
  • <span>: Another generic container, but this one is for small pieces *inline* with text (e.g., styling a single word).

Example combining several tags:

<h2>My Favorite Fruits</h2>
<p>Here is a list of fruits I enjoy:</p>
<ul>
  <li>Apples</li>
  <li>Bananas</li>
  <li>Strawberries</li>
</ul>
<p>Learn more at <a href="https://example.com" target="_blank">Example Fruit Site</a>.</p>
<!-- The target="_blank" attribute opens the link in a new tab -->
<img src="../images/placeholder.png" alt="Placeholder Image" width="100">
<!-- Assuming a placeholder image exists at ../images/placeholder.png -->
<!-- The width attribute sets the image width in pixels -->

Giving Tags More Information: HTML Attributes

Attributes are extra bits of information added *inside* the opening tag of an element. They usually come in name="value" pairs and tell the browser more about the element or how it should behave.

You've already seen some:

  • href="url": Used with the <a> tag to specify the link's destination (Hypertext Reference).
  • src="filename": Used with the <img> tag to specify the path to the image file (Source).
  • alt="description": Used with the <img> tag to provide alternative text. This is important for screen readers (used by visually impaired users) and also displays if the image can't load.
  • width="pixels" / height="pixels": Can be used on images (and other elements) to suggest dimensions.
  • target="_blank": Used on an <a> tag to make the link open in a new browser tab.
  • title="tooltip text": Provides extra advisory information, often shown as a tooltip when hovering over the element.

Two crucial attributes used heavily with CSS are:

  • class="name": Assigns one or more class names to an element. You can give the *same* class name to multiple elements. Think of it like putting a category label on elements (e.g., class="important-warning"). Used to apply the same style to many elements.
  • id="unique_name": Assigns a unique identifier to *one specific* element on the page. An ID must be unique â€“ no two elements on the same page can have the same ID. Think of it like giving an element a specific name (e.g., id="main-navigation-bar"). Used to target one specific element for styling or JavaScript.
<!-- An element can have multiple classes, separated by spaces -->
<div class="highlight warning-box">This div has two classes.</div>

<!-- An ID must be unique on the page -->
<p id="introduction">This paragraph has a unique ID.</p>

<a href="https://google.com" title="Visit Google Search" target="_blank">Go to Google</a>

Choosing between class and id: Use class when you might want to style multiple elements the same way. Use id only when you need to uniquely identify one specific element.

Common CSS Selectors (More Detail)

  • Element Selector: p { color: gray; } (Styles all <p> elements).
  • Class Selector: .important { font-weight: bold; } (Styles any element with class="important"). You can chain classes: .important.highlight { background-color: yellow; } (Styles elements with *both* classes).
  • ID Selector: #main-logo { border: 1px solid black; } (Styles the single element with id="main-logo"). Use IDs sparingly, primarily for unique page landmarks or JavaScript hooks.
  • Attribute Selector: a[target="_blank"] { background: lightblue; } (Styles links that open in a new tab).
  • Descendant Combinator: nav ul { list-style: none; } (Styles only <ul> elements that are *inside* a <nav> element).
  • Pseudo-classes: Select based on state, like a:hover { text-decoration: none; } (Styles links when the mouse is over them). We'll cover more in Chapter 3.

Example combining selectors:

/* Style all paragraphs inside elements with class "article" */
.article p {
  line-height: 1.8;
}

/* Style the element with ID "submit-button" when hovered */
#submit-button:hover {
  background-color: green;
  color: white;
}

Common CSS Properties

There are hundreds of CSS properties. Here are a few basics:

  • color: Sets the text color.
  • background-color: Sets the background color of an element.
  • font-size: Sets the size of the text.
  • font-family: Sets the typeface (e.g., Arial, 'Times New Roman', sans-serif).
  • margin: Controls the space *outside* an element's border.
  • padding: Controls the space *inside* an element's border (between the border and the content).
  • border: Shorthand for setting border width, style (solid, dashed, dotted), and color. Example: border: 2px dashed red;
  • width / height: Set the dimensions of an element (doesn't apply to all element types).
  • text-align: Aligns text within an element (left, center, right, justify).

Next Steps & Practice

  • Experiment: Modify the code on this page or create your own simple HTML file. Try changing colors, fonts, and adding new tags.
