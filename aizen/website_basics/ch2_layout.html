Chapter 2: Layout

In this chapter, we'll explore different ways to layout your web pages using CSS Flexbox and Grid. These powerful tools will help you create responsive and beautiful designs.

Why Do We Need Layout Techniques?

Imagine trying to build a webpage with a navigation bar at the top, a sidebar on the left, and main content on the right. Just letting the elements flow naturally won't achieve this structure. CSS layout tools let us override the default flow.

We will focus on the two most important modern techniques:

  • Flexbox: Excellent for arranging items in a single direction (either a row OR a column). Great for navigation bars, aligning items in a component, distributing space.
  • Grid: Excellent for arranging items in two directions (rows AND columns) at the same time. Perfect for overall page layouts, complex dashboards, or galleries.

We'll also briefly mention older methods you might see in existing code, but Flexbox and Grid are the tools you should learn and use for new projects.

Flexbox Layout

Flexbox is perfect for one-dimensional layouts - either rows or columns. It's great for:

  • Navigation bars
  • Card layouts
  • Centering content
  • Equal-height columns

CSS Grid Layout

Grid is perfect for two-dimensional layouts - organizing content in both rows and columns. Use it for:

  • Page layouts
  • Image galleries
  • Complex dashboard layouts
  • Any design that needs precise control over rows and columns

Older Layout Methods (Briefly)

While Flexbox and Grid are the modern standards, you might see older techniques in existing websites:

  • Float: Originally meant for wrapping text around images, floats were used for layouts before Flexbox/Grid.
  • Position: Using absolute/relative positioning for specific layout needs.
  • Tables: (Not recommended) Very old websites sometimes used tables for layout.

Focus on learning Flexbox and Grid - they're the future of web layout!
